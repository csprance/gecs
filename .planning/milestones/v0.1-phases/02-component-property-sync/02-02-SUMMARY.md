## What Was Built

**CN_NetSync** (`addons/gecs_network/components/cn_net_sync.gd`) is the new per-entity network sync component that:

- Holds a `Priority` enum (REALTIME=0, HIGH=1, MEDIUM=2, LOW=3)
- Defines `PRIORITY_MAP` mapping @export_group name strings to priority ints (SPAWN_ONLY → -2, LOCAL → -1 are sentinels excluded from cache)
- Provides `scan_entity_components(entity)`: iterates `entity.components`, skips itself (CN_NetSync) and CN_NetworkIdentity, calls `_scan_component(comp)` per sibling, builds `_cache_by_comp / _props_by_comp / _comp_refs / _comp_type_names` keyed by `comp.get_instance_id()`
- Provides `check_changes_for_priority(priority: int) -> Dictionary`: polls changed props at given priority, updates cache on detection, returns wire-format `{ "CompTypeName": { "prop_name": value } }`
- Provides `update_cache_silent(comp, prop, value)`: brings cache in sync without triggering change detection (prevents echo loops in SyncReceiver)
- Ports `_has_changed()` and `_deep_copy()` verbatim from deleted `sync_component.gd`

**sync_config.gd** was stripped to a minimal stub (6 lines) with only the fields that Phase 3/4 handler files still reference.
