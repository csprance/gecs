Verified save/load cycle:
```gdscript
func save_game(slot: int) -> void:
    var q = ECS.world.query.with_all([C_Persistent])
    var data = ECS.serialize(q)
    ECS.save(data, "user://saves/slot_%d.tres" % slot, true)  # binary=true -> .res format

func load_game(slot: int) -> void:
    ECS.world.purge()
    var entities = ECS.deserialize("user://saves/slot_%d.tres" % slot)
    for entity in entities:
        ECS.world.add_entity(entity)
```

Selective serialization with config:
```gdscript
var config = GECSSerializeConfig.new()
config.include_all_components = false
config.components = [C_Health, C_Inventory]
config.include_relationships = true
config.include_related_entities = false
var data = ECS.serialize(ECS.world.query.with_all([C_Player]), config)
```

World-level config:
```gdscript
# Set default config on the World node (Inspector or code)
ECS.world.default_serialize_config = GECSSerializeConfig.new()
# Per-entity: entity.serialize_config overrides world default for that entity
```

Key facts:
- Relationships ARE serialized (include_relationships: bool = true is the default)
- include_related_entities pulls in entities that are targets of relationships
- binary=false -> saves as .tres (text), binary=true -> saves as .res (binary)
