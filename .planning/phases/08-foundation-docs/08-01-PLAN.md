Minimal complete ECS loop (verified):
```gdscript
# main.gd
extends Node
@onready var world: World = $World
func _ready():
    ECS.world = world
    var entity = Entity.new()
    ECS.world.add_entity(entity, [C_Health.new(100), C_Velocity.new()])
func _process(delta):
    ECS.process(delta)
```

Scene-based entity (spatial — requires Node3D root):
```gdscript
class_name Player
extends Entity
func define_components() -> Array:
    return [C_Health.new(100), C_Velocity.new()]
func on_ready():
    # Only works because this scene root is Node3D
    var c_vel = get_component(C_Velocity) as C_Velocity
    if c_vel:
        c_vel.direction = Vector3.RIGHT
```

Code-based entity (pure data, no spatial):
```gdscript
class_name GameTimer
extends Entity
func define_components() -> Array:
    return [C_Timer.new(30.0)]
```

add_entity signature: world.add_entity(entity: Entity, components = null, add_to_tree = true)
- Do NOT call add_child() before add_entity() — world manages tree placement
- OR pass add_to_tree=false if manually managing the scene tree

ECS.process(delta, group="") — second arg optional
Entity extends Node — NOT Node3D. global_position only exists when scene root is Node3D/Node2D.
SystemGroup is NOT a GECS class — use Node for system parent nodes in scene diagrams.
CommandBuffer: cmd.remove_entity(entity) — available on System as self.cmd property.
